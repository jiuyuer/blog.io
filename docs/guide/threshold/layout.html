# 布局

页面整体布局是一个产品最外层的框架结构，往往会包含头部，导航、侧边栏、面包屑以及内容等。想要了解一个后台项目，先要了解它的基础布局。

# 组件概述

  • Layout： 布局容器，其下可嵌套 Topbar，Sidebar，App-main，一般用来作为全局配置。
  • Topbar： 顶部布局，头部容器，其下可嵌套任何元素，只能放在 Layout 中。
  • Sidebar： 侧边栏，自带默认样式及基本功能，其下可嵌套任何元素，只能放在 Layout 中。
  • App-main： 内容部分，自带默认样式，其下可嵌套任何元素，只能放在 Layout 中。

# Layout

# layout.vue 的代码结构如下

<!-- layout.vue -->
<template>
  <div class="app-wrapper" :class="{hideSidebar:!sidebar.opened}">
    <div class="view-base-bar">
      <topbar></topbar>
      <hamburger class="sidebar-fold" :toggleClick="toggleSideBar" :isActive="sidebar.opened"></hamburger>
      <sidebar class="sidebar-container"></sidebar>
    </div>
    <div class="view-framework">
      <div class="main-container">
        <breadcrumb></breadcrumb>
        <app-main></app-main>
      </div>
    </div>
  </div>
</template>

项目中大部分页面都是基于这个 layout 模板构成的，除了个别页面如： login , 404, 401 等页面没有使用该 layout。如果你想在一个项目中有多种不同的 layout 也是很方便的，只要在一级路由那里选择不同的 layout 组件就行。

//路由文件

//引入Layout
import Layout from '@/views/layout/Layout';
//引入其他的Layout
import Blank from '@/views/layout/Blank';

// No layout
{
  path: '/login',
  component: () =>
    import ('@/views/login/index')
},

// Has layout
{
  path: '/dashboard',

  //你可以选择不同的layout组件
  component: Layout,

  //这里开始对应的路由都会显示在<app-main></app-main>中 layout.vue中可查看
  children: [{
    path: 'dashboard',
    name: 'dashboard',
    component: () =>
      import ('@/views/dashboard/index')
  }]
}

这里使用了 vue-router 路由嵌套, 所以一般情况下，你增加或者修改页面只会影响 app-main 这个主体区域。其它配置在 layout 中的内容如：侧边栏或者导航栏都是不会随着你主体页面变化而变化的。

/foo                                  /bar
+------------------+                  +-----------------+
| layout           |                  | layout          |
| +--------------+ |                  | +-------------+ |
| | foo.vue      | |  +------------>  | | bar.vue     | |
| |              | |                  | |             | |
| +--------------+ |                  | +-------------+ |
+------------------+                  +-----------------+

当然你也可以一个项目里面使用多个不同的 layout，只要在你想作用的路由父级上引用它就可以了。

# app-main

<template>
  <section class="app-main">
    <transition name="fade-transform" mode="out-in">
      <keep-alive :include="cachedViews">
        <router-view :key="key"></router-view>
      </keep-alive>
    </transition>
  </section>
</template>
  • app-main 是系统的主体区域，几乎所有的业务功能模块开发都是在这里面渲染展示的
  • app-main 文件的代码片段中，为了缓存 <router-view>，在其外部包了一层 keep-alive。
  • 其中 transition 标签定义了页面之间切换动画，可以根据自己的需求，自行修改转场动画。

# router-view

<router-view> 组件是一个 functional 组件，渲染路径匹配到的视图组件。<router-view> 渲染的组件还可以内嵌自己的 <router-view>，根据嵌套路径，渲染嵌套组件。
